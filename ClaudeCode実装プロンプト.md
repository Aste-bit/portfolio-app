# ポートフォリオ実装プロンプト

このフォルダ（portfolio-app）のVite + Reactプロジェクトで、ポートフォリオサイトを実装してください。

## プロジェクト状況

- Vite + React + Tailwind CSS v4 はセットアップ済み（ビルド確認済み）
- 設計書 `ポートフォリオ設計書_完全版.md` に全仕様が記載されています
- **まず設計書を読んでから作業を開始してください**

## 最初にやること

```bash
npm install framer-motion lucide-react
```

## コンセプト

**「静かにすごい」**— 自慢しない、アピールしない、丁寧に見せる。
余白をたっぷり使い、読み物のようなゆったりした体験にする。
アニメーションは控えめ（fadeIn + 微かなスライドアップのみ）。

## 守ること（厳守）

- 設計書のカラーシステム（Stone系ウォームグレー）を忠実に
- タイポグラフィの指定を忠実に
- テキスト内容は設計書のコンテンツデータをそのまま使う（勝手に変えない）
- 余白は多めに。詰め込まない
- `max-w-3xl`（狭め）で読み物感を出す
- スクリーンショットは `/public/screenshots/` から読み込み、画像がなくてもエラーにならないプレースホルダーを表示
- ダーク/ライト切り替え（localStorage + system preference）
- モード切り替え（わかりやすく / 技術詳細）+ `?mode=tech` URLパラメータ対応
- レスポンシブ（デスクトップ3列 / タブレット2列 / モバイル1列）

## 自由にしてよいこと

- コンポーネントの分割粒度
- Tailwindクラスの具体的な書き方
- アニメーションのduration微調整
- アコーディオンの実装方法

## 実装の進め方

設計書に記載の優先順位順に、1セクションずつ作ってください：

1. `data/content.js` — 全テキストデータをまとめる
2. `hooks/`（useTheme, useMode）— グローバル状態
3. `FadeIn.jsx` — アニメーション共通ラッパー
4. `Navbar` — sticky, backdrop-blur, トグル2つ
5. `Intro` — ゆっくりフェードイン、テキスト切り替え
6. `Project` — 説明文 + スクショ枠 + タグ + アコーディオン（技術版のみ）
7. `Capabilities` — カードグリッド + テキスト追記
8. `Process` — フロー図 + アコーディオン（技術版のみ）
9. `Footer`
10. 全体の磨き上げ — 余白調整、フォント微調整、ダークモード確認

**各セクション完成ごとに `npm run dev` で動作確認しながら進めてください。**

## 完了条件

- `npm run build` が通ること
- ライト/ダークモード両方でデザインが崩れないこと
- わかりやすく/技術詳細モードの切り替えが動くこと
- モバイル表示で問題がないこと
- `?mode=tech` でアクセスした時に技術版が表示されること
